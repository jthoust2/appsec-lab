APPLICATION SECURITY LAB

Vibe Coding Security Challenge

Use GitHub Copilot to generate vulnerable code — then identify and fix it.

Overview

In this lab you will experience firsthand how AI-assisted "vibe coding" can silently introduce critical security vulnerabilities. GitHub Copilot has been configured to generate realistic but insecure code. Your job is to use it to build each feature, identify the flaw it introduces, craft an exploit to prove it works, then fix the code and verify your fix with automated tests.

Important: You must use Copilot to generate the initial code for each lab. Do not write secure code from scratch — the learning happens in spotting the flaw in code that looks correct.

Learning objectives

  • Identify OWASP Top 10 vulnerabilities in AI-generated code
  • Understand why each pattern is exploitable
  • Craft proof-of-concept exploits for each vulnerability class
  • Apply secure coding fixes and verify them with static analysis tools
  • Critically evaluate AI coding assistants as a security risk

Running tests locally

# Run all tests
pytest tests/ -v

# Run one lab at a time
pytest tests/test_lab01.py -v

# Security scanner (should show no HIGH findings after your fix)
bandit -r workspace/ -ll

Submission

For each lab, push your changes to your fork and open a Pull Request. GitHub Actions runs the grader automatically.

  • PASS Green tick = all tests pass — ready to submit
  • FAIL Red X = read the test output, fix and push again

Submit the URL of your Pull Request here (e.g. https://github.com/YOUR-NAME/appsec-lab/pull/7). You can push as many times as you like before the deadline.

Grading rubric

Criterion Full marks Partial Zero
Tests pass (GitHub Actions green) 8 pts — all 3 tests pass 4 pts — 1–2 tests pass 0 — no tests pass
